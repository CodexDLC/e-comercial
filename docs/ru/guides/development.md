# Процесс разработки 🧪

Мы поддерживаем высокие стандарты качества кода с помощью автоматизированных инструментов и строгих проверок.

## 🛡️ Контроль качества (Quality Gate)
В проекте настроен единый скрипт для запуска всех инструментов проверки качества:
```bash
python tools/dev/check.py
```
Скрипт выполняет следующие действия:
1. **Линтинг**: `ruff` для проверки стиля кода.
2. **Типизация**: `mypy` для статического анализа типов.
3. **Безопасность**: `bandit` для поиска уязвимостей.
4. **Секреты**: `detect-secrets` для поиска утечек учетных данных.
5. **Тесты**: `pytest` для функциональной верификации.

## 🧪 Тестирование
Для написания тестов используется `pytest`.
```bash
# Запуск всех тестов
pytest

# Запуск с отчетом о покрытии
pytest --cov=src --cov-report=html
```

## 📐 Миграции
При изменении моделей всегда используйте стандартный рабочий процесс Django:
```bash
python src/Hop-and-Barley/manage.py makemigrations
python src/Hop-and-Barley/manage.py migrate
```

## 🎨 Управление ассетами
Статические файлы (CSS/JS) управляются через `codex-django`:
- `python manage.py dev`: Автоматическая сборка ассетов при разработке.
- `python manage.py compile_assets`: Сборка ресурсов для продакшена.
