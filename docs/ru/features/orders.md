# Заказы и Корзина 🛒

Модуль заказов управляет процессом от добавления товара в корзину до финального оформления.

## 🛒 Корзина на сессиях
Мы используем реализацию корзины на основе сессий (`features/orders/cart.py`). Это позволяет пользователям сохранять товары без необходимости входа в аккаунт.
- **Хранение**: Данные сериализуются в JSON и хранятся в сессии пользователя.
- **Контроль остатков**: Метод `Cart.add()` автоматически ограничивает количество товара на основе доступных остатков на складе.

## 🧾 Оформление заказа
1. **Наполнение корзины**: Товары добавляются в сессионную корзину.
2. **Checkout**: Пользователь указывает данные для доставки и подтверждает заказ.
3. **Бизнес-логика**: Сервис `features/orders/services/order.py` создает записи `Order` и `OrderItem`.
4. **Списание остатков**: При успешном оформлении заказа количество товара на складе уменьшается.

## 📊 Кабинет клиента
Клиенты могут отслеживать историю своих заказов через личный кабинет (`space="client"`), который работает на базе системы кабинетов `codex-django`.
