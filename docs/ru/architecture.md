# Архитектура проекта 🏗

Hop and Barley использует модульную архитектуру, обеспечивающую масштабируемость и простоту поддержки, с использованием библиотеки `codex-django`.

## Структура компонентов

- **`core/`**: Центральная конфигурация: настройки, основные URL-адреса, логирование и инфраструктура (Redis).
- **`features/`**: Бизнес-логика, сгруппированная по доменам. Каждая фича содержит свои модели, сервисы, селекторы и API.
- **`system/`**: Общие сервисы и системная логика, включая систему **Кабинетов**.
- **`cabinet/`**: Шаблоны и компоненты для интерфейсов Staff (Сотрудники) и Client (Клиенты).

## Ключевые паттерны разработки

### 1. Паттерн Service/Selector
Для разделения логики мы используем строгое разграничение:
- **Selectors**: Отвечают за получение данных и сложные фильтрации. Возвращают QuerySet или словари, никогда не изменяют данные.
- **Services**: Отвечают за бизнес-логику и изменение данных. Любое изменение состояния (создание заказа, обновление остатков) должно проходить через Сервис.

### 2. Система Кабинетов
Проект использует модуль `cabinet` для управления интерфейсами:
- **`declare()`**: Регистрация модулей, пунктов меню и виджетов для разных пространств (`staff` vs `client`).
- **Виджеты дашборда**: Графики (KPI), списки и отчеты регистрируются и настраиваются через систему кабинетов.

## Поток данных
1. **Запрос**: Попадает во View или DRF ViewSet.
2. **Логика**: View вызывает **Service** для изменений или **Selector** для получения данных.
3. **Ответ**: Данные передаются в шаблон (с использованием HTMX/Alpine.js для интерактивности) или сериализуются в JSON.
